# EDINETデータ構造の詳細分析レポート

## 概要

このレポートは、`edinet2dataset`モジュールで取得できるデータ構造を、実際のトヨタ自動車株式会社（EDINETコード: E02144）の有価証券報告書をサンプルとして詳しく分析したものです。

## 1. TSVファイルの基本構造

### ファイル情報
- **ファイル**: `data/E02144/S100TR7I.tsv`
- **総行数**: 2,253行
- **エンコーディング**: UTF-16
- **区切り文字**: タブ（TSV形式）

### カラム構成
TSVファイルには以下の9つのカラムが含まれています：

1. **要素ID**: XBRLの要素識別子（例: `jpcrp_cor:NetSalesSummaryOfBusinessResults`）
2. **項目名**: 日本語の項目名（例: `売上高、経営指標等`）
3. **コンテキストID**: 年度や時点を表す識別子（例: `CurrentYearDuration`）
4. **相対年度**: 相対的な年度表現（例: `当期`, `前期`, `前々期`）
5. **連結・個別**: 連結決算か個別決算かの区分（`その他`=連結、`個別`=個別）
6. **期間・時点**: 期間データか時点データか（`期間` or `時点`）
7. **ユニットID**: 単位の識別子（例: `JPY`, `pure`, `JPYPerShares`）
8. **単位**: 単位の表示名（例: `円`, `－`）
9. **値**: 実際の数値またはテキストデータ

## 3. 年度データの構造

### 3.1 年度キーの種類

パーサーは以下の年度キーを認識します：

- `Prior4Year`: 4期前
- `Prior3Year`: 3期前
- `Prior2Year`: 前々期
- `Prior1Year`: 前期
- `CurrentYear`: 当期
- `Prior1YTD`: 前期累計
- `CurrentYTD`: 当期累計
- `Prior1Quarter`: 前期四半期
- `CurrentQuarter`: 当期四半期
- `FilingDate`: 提出日

### 3.2 コンテキストIDと年度の対応

- **Duration**: 期間データ（損益計算書、キャッシュフロー計算書）
  - `CurrentYearDuration` → `CurrentYear`
  - `Prior1YearDuration` → `Prior1Year`
  - など

- **Instant**: 時点データ（貸借対照表）
  - `CurrentYearInstant` → `CurrentYear`
  - `Prior1YearInstant` → `Prior1Year`
  - など

### 3.3 データの有無

年度によってデータの有無が異なります：

- **CurrentYear**: 最も多くのデータが存在（1,236行）
- **Prior1Year**: 次に多い（632行）
- **Prior2Year**: 一部のデータのみ（65行）
- **Prior3Year**: 限定的（41行）
- **Prior4Year**: 最も少ない（59行）

## 4. データの特徴と制約

### 4.1 連結決算のみ対応

現在のパーサーは連結決算のみを抽出します。個別決算のデータは除外されます。

### 4.2 要素IDのマッピング

`element_id_table.py`で定義された要素IDマッピングに基づいて、XBRLの要素IDを日本語の項目名に変換します。マッピングにない要素IDは抽出されません。

### 4.3 データの型

- **数値データ**: 文字列として格納（例: `"45095325000000"`）
- **テキストデータ**: 文字列として格納
- **単位**: 円、純数値、1株当たりなど

### 4.4 欠損データ

年度によってデータが存在しない場合、その年度のキーは辞書に含まれません。

## 5. 使用例

# 基本情報の取得
company_name = financial_data.meta["会社名"]

operating_profit = financial_data.pl["営業利益"]

## 6. まとめ

`edinet2dataset`モジュールは、EDINETのXBRLデータを以下のように構造化します：

1. **META**: 企業・書類の基本情報（年度情報なし）
2. **SUMMARY**: 主要財務指標（年度別）
3. **BS**: 貸借対照表の詳細項目（年度別、時点データ）
4. **PL**: 損益計算書の詳細項目（年度別、期間データ）
5. **CF**: キャッシュフロー計算書の詳細項目（年度別、期間データ）
6. **TEXT**: テキストブロック（年度別）

各カテゴリは辞書形式で、年度情報を持つ場合はネストされた辞書構造になります。これにより、Pythonで簡単に財務データを分析・処理できます。
