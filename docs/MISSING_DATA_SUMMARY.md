# サンプルデータで足りていないパラメーター・数値のまとめ

フロント（企業一覧テーブル・企業分析ページ）で「－」になりやすい理由を、**現行実装**（`build_screener_data.py`・補完ロジックあり）に合わせて整理します。

詳細なコマンド・列と EDINET の対応は [edinet-wrapper-使い方.md](./edinet-wrapper-使い方.md) を参照してください。

---

## 1. 株価・時価が必要で、現状は常に埋まらない項目

`build_screener_data.py` では **株価 API を呼ばない**ため、次のキーは **意図的に未計算（null）** です。将来、株価ソースを繋いだときに同じキーへ書き込む想定です。

| フロント表示名 | データキー | 説明 |
|----------------|------------|------|
| **PBR** | `PBR` | 株価 ÷ BPS。BPS は開示から取れるが **株価が無い** |
| **時価総額** | `時価総額` | 株価 × 発行済株式数 |
| **ネットキャッシュ比率** | `ネットキャッシュ比率` | ネットキャッシュ ÷ 時価総額の設計のため **時価が必要** |

→ **対応方針**: 株価 API 連携後に計算・格納。

**注意（過去ドキュメントとの差分）**: PER・配当利回りは、開示データと補完ロジックにより **null でない場合があります**（下記）。

---

## 2. EDINET のキー有無・期の取り方で欠けやすい項目

### 2.1 開示ベースの PER / ROE / BPS など

- **四半期・半期**の summary には、株価収益率・自己資本利益率・BPS が **載らないことが多い**。
- **対策（実装済み）**: 上記キーは **最新期に無ければ、新しい期から遡って**各キー単位で補完する。  
  → **データが存在すれば** `company_metrics` に入る。全期で欠けると null。
- **注意**: 補完元は **必ずしも「計算日」と同じ決算期ではない**（最新は半期・PER は前年有報、など）。

### 2.2 その他、キーが無い・PL が空で欠けやすいもの

| フロント表示名 | データキー | 不足しやすい理由 |
|----------------|------------|-------------------|
| **営業利益** | `営業利益` | `pl` が空の提出（半期など）で欠ける |
| **発行済株式総数** | `発行済株式総数` | summary に無い期がある |
| **営業 CF 等** | `営業CF` / `投資CF` / `財務CF` | summary と `cf` のキー名差で片方のみ |
| **配当性向・DPS（開示）** | `配当性向` / `dividendPerShare` | 四半では未掲載のことがある（補完キーに含むが、全期で無ければ null） |

→ **対応方針**: パーサーでのキー別名対応、または欠損時は算出列（`payoutRatioComputed` 等）を参照。

### 2.3 企業によって「決算サマリー一式」が null

`data-set` 内の TSV の **期の取り方**（CurrentYear 等）や **構造差**により、特定企業だけ `summary` の売上・EPS 等が取れない場合があります。原因切り分けは **該当企業の `summaries/{secCode}.json`** と **data_quality_report** で確認してください。

---

## 3. `companies.json` と `summaries` の件数差

- **`companies.json`**: 最後に実行した `build_screener_data.py --mode sample` の **対象 EDINET コードだけ**（例: 11 社）。
- **`summaries/*.json`**: 過去のビルドで残った銘柄が残ることがあり、**ファイル数 > 一覧の社数**になり得ます。

一覧に出したい範囲は **`--list` または引数で揃えたうえで**フルビルドし、不要な `summaries` を手で削除するか、出力先を空にしてから再生成する運用が確実です。

---

## 4. 分析ページの指標表示

`/analyze/[secCode]` の指標テーブルは `company_metrics` の 1 行を表示します。欠損は **データが null** か **型表示の都合**で「－」になります。

---

## 5. チェックリスト（優先度の目安）

| 優先度 | 内容 |
|--------|------|
| 高 | 株価 API → PBR・時価総額・ネットキャッシュ比率 |
| 中 | 特定企業で summary が空になる TSV 構造の調査 |
| 中 | キー別名のマッピング拡張（CF・現金など） |
| 低 | `companies.json` と `summaries` の対象を一致させる運用 |
