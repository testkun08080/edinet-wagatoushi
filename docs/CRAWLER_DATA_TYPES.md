# クローラーで取得できるデータの種類

このドキュメントは、edinet-wrapper のクローラー（Downloader + パーサー）で取得・利用できるデータの種類を一覧にしたものです。

---

## 1. 取得元の概要


| 取得元                   | 説明                                     | 用途                        |
| --------------------- | -------------------------------------- | ------------------------- |
| **EDINET 書類一覧 API**   | 日付指定で提出書類のメタデータ一覧を取得                   | 書類リスト・docID の取得           |
| **Edinetcode（企業マスタ）** | 提出者一覧 CSV（ZIP 配布）                      | 企業名 ↔EDINET コード・業種・証券コード等 |
| **書類本体 API**          | docID 指定で書類を PDF / XBRL / TSV(CSV) で取得 | 有報本文・XBRL 数値・TSV パース用     |


---

## 2. 書類一覧 API（documents.json）で取得できるデータ

`get_results(start_date, end_date, edinet_code?)` で取得する **Result** 1 件あたりの項目です。


| 項目                   | 説明                |
| -------------------- | ----------------- |
| seqNumber            | 連番                |
| docID                | 書類 ID（ダウンロード時に使用） |
| edinetCode           | EDINET コード        |
| secCode              | 証券コード             |
| JCN                  | 法人番号              |
| filerName            | 提出者名              |
| fundCode             | ファンドコード           |
| ordinanceCode        | 府令コード             |
| formCode             | 様式コード             |
| docTypeCode          | 書類種別コード           |
| periodStart          | 対象期間開始日           |
| periodEnd            | 対象期間終了日           |
| submitDateTime       | 提出日時              |
| docDescription       | 書類の概要・備考          |
| issuerEdinetCode     | 発行者 EDINET コード    |
| subjectEdinetCode    | 対象 EDINET コード     |
| subsidiaryEdinetCode | 子会社 EDINET コード    |
| currentReportReason  | 臨時報告書の理由          |
| parentDocID          | 親書類 ID            |
| opeDateTime          | 運用日時              |
| withdrawalStatus     | 取下げ有無             |
| docInfoEditStatus    | 書類情報編集状況          |
| disclosureStatus     | 開示状況              |
| xbrlFlag             | XBRL 有無           |
| pdfFlag              | PDF 有無            |
| attachDocFlag        | 添付書類有無            |
| englishDocFlag       | 英文有無              |
| csvFlag              | CSV(TSV) 有無       |
| legalStatus          | 法令上のステータス         |


**書類種別**（`ordinanceCode` + `formCode` から判定）の例: 有価証券報告書(annual)、訂正有報(annual_amended)、四半期報告書(quarterly)、四半期訂正(quarterly_amended)、半期報告書(semiannual)、半期訂正(semiannual_amended) など。

## 3. Edinetcode（企業マスタ）で取得できるデータ

`download_edinetinfo_csv()` で取得する **EdinetcodeDlInfo.csv** のカラムです。


| カラム名      | 説明                  |
| --------- | ------------------- |
| ＥＤＩＮＥＴコード | 企業・ファンドの EDINET コード |
| 提出者種別     | 提出者の種別              |
| 上場区分      | 上場・マザーズ等            |
| 連結の有無     | 連結決算の有無             |
| 資本金       | 資本金                 |
| 決算日       | 決算日                 |
| 提出者名      | 会社名（日本語）            |
| 提出者名（英字）  | 会社名（英語）             |
| 提出者名（ヨミ）  | よみがな                |
| 所在地       | 本店所在地               |
| 提出者業種     | 業種（EDINET の業種分類）    |
| 証券コード     | 証券コード（上場銘柄）         |
| 提出者法人番号   | 法人番号                |


企業名検索（`search_company`）では「提出者名」「ＥＤＩＮＥＴコード」「提出者業種」を主に利用します。

## 4. 書類本体の取得形式

`download_document(doc_id, file_type, output_dir)` で指定できる **file_type** と取得内容です。


| file_type | 説明            | 取得物                                            |
| --------- | ------------- | ---------------------------------------------- |
| pdf       | 有価証券報告書等の PDF | 1 ファイル（docID.pdf）                              |
| xbrl      | XBRL パッケージ    | ZIP 解凍後の XBRL ファイル群                            |
| tsv       | TSV（実体は CSV）  | XBRL から変換した `jpcrp*.csv` を `.tsv` として保存（パーサー用） |


---

## 5. TSV パース後に利用できるデータ（FinancialData）

TSV を `parse_tsv()` すると **FinancialData** が得られます。中身は次の 6 カテゴリです。

### 5.1 META（基本情報・年度なし）

企業・書類の 1 報告書あたり 1 セットの情報です。


| 項目（例）                            | 説明            |
| -------------------------------- | ------------- |
| 会社名                              | 提出者名          |
| EDINET コード                       | EDINET コード    |
| 証券コード                            | 証券コード         |
| 提出書類                             | 有価証券報告書等      |
| 会計基準                             | 日本基準 / IFRS 等 |
| 当事業年度開始日・終了日                     | 対象事業年度        |
| 連結決算の有無                          | true / false  |
| 修正の有無 / 記載事項訂正のフラグ / XBRL 訂正のフラグ | 訂正有無          |


### 5.2 SUMMARY（業績サマリー・年度別）

主要な経営指標を年度キー（CurrentYear, Prior1Year 等）で参照できます。

- 売上高、経常利益、税引前利益、親会社株主に帰属する当期純利益、当期純利益又は当期純損失
- 包括利益、純資産額、総資産額
- 1 株当たり純資産額、1 株当たり当期純利益、潜在株式調整後 1 株当たり当期純利益
- 自己資本比率、自己資本利益率、株価収益率
- 営業・投資・財務キャッシュフロー、現金及び現金同等物の残高
- 従業員数、平均臨時雇用人員
- 配当性向、1 株当たり中間配当額・配当額、発行済株式総数、資本金 など

### 5.3 BS（貸借対照表・年度別・時点データ）

資産・負債・純資産の各項目を年度別に取得できます。

- 流動資産: 現金及び預金、現金及び現金同等物、売掛金、有価証券、商品及び製品 等
- 固定資産: 有形固定資産、無形固定資産、投資その他 等
- 流動負債・非流動負債
- 純資産: 株主資本、その他の包括利益累計額、非支配株主持分 等

（項目は企業・会計基準により増減します。）

### 5.4 PL（損益計算書・年度別・期間データ）

売上から当期純利益までの損益項目を年度別に取得できます。

- 売上高、売上原価、売上総利益
- 販売費及び一般管理費、営業利益
- 営業外収益・費用、経常利益
- 特別損益、税引前利益、法人税等、当期純利益 等

### 5.5 CF（キャッシュフロー計算書・年度別・期間データ）

営業・投資・財務キャッシュフローと関連項目を年度別に取得できます。

- 当期利益、減価償却費及び償却費
- 営業キャッシュフロー、投資キャッシュフロー、財務キャッシュフロー
- 現金及び現金同等物の増減額、現金及び現金同等物 等

### 5.6 TEXT（テキストブロック・年度別）

有報の長文テキストを年度別に取得できます。

- 沿革、事業の内容、関係会社の状況、従業員の状況
- 業績等の概要、生産・受注・販売の状況
- 経営方針・経営環境・対処すべき課題等、事業等のリスク
- 経営上の重要な契約等、研究開発活動
- 財政状態・経営成績・キャッシュフローの状況の分析
- 設備投資等の概要、主要な設備の状況、設備の新設・除却等の計画
- 株式の総数、発行済株式・株式の総数等、発行済株式総数・資本金等の推移
- 所有者別状況、大株主の状況、発行済株式・議決権の状況、自己株式等、株式の種類等
- 配当政策、株価（5 年間・6 か月）、役員の状況、コーポレート・ガバナンス 等

---

## 6. 年度キー（パース後の辞書で使用）

SUMMARY / BS / PL / CF / TEXT の年度別データで使われるキーの例です。

- Prior4Year, Prior3Year, Prior2Year, Prior1Year, CurrentYear
- Prior1YTD, CurrentYTD（累計）
- Prior1Quarter, CurrentQuarter（四半期）
- FilingDate（提出日）

企業・書類によって存在する年度は異なります（CurrentYear / Prior1Year が主）。

---

## 7. 注意事項

- **連結のみ**: パーサーは連結決算のデータを抽出します。個別決算のみの項目は別途必要なら TSV 生データや element_id_table の拡張が必要です。
- **要素 ID マッピング**: パースで使う項目は `element_id_table.py` の META / SUMMARY / BS / PL / CF / TEXT に定義された要素に限られます。未定義の要素は FinancialData には含まれません。
- **数値の型**: パース後の数値は文字列で保持されます（例: `"45095325000000"`）。計算時は整数・float への変換が必要です。
- **書類種別**: 有価証券報告書を前提にした TSV パースです。四半期・半期報告書は様式が異なる場合があります。
